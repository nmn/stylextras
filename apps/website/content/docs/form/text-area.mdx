# Text Area

Use Text Area inside forms where native input behavior, labeling, validation, and token-driven styling need to stay aligned.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

Search aliases: textarea, multiline input, multiline field, message field.

## Import

```tsx
import { TextAreaField, TextArea } from "@stylextras/ui/text-area";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

## Props

The public prop types are exported from `@stylextras/ui/text-area`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type TextAreaProps = Omit<
  BaseProps,
  "aria-label" | "aria-labelledby" | "className" | "style"
> &
  AccessibleNameProps & {
    labelSx?: StyleXStyles;
    size?: TextareaSize;
    sx?: StyleXStyles;
    textareaSx?: StyleXStyles;
  };

export type TextAreaFieldProps = TextAreaProps;
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { TextArea } from "@stylextras/ui/text-area";

function TextAreaExample() {
  return (
    <>
      <div>
        <TextArea
          label="Summary"
          rows={4}
          defaultValue="Short multiline content."
        />
        <TextArea
          label="Notes"
          rows={6}
          defaultValue="Longer multiline content helps show the rhythm, spacing, and line height of the component without extra layout noise."
        />
      </div>
    </>
  );
}
```

## Accessibility notes

Every interactive form control should have a label or accessible name. Preserve native attributes such as required, disabled, min, max, and aria-describedby when they apply.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/text-area/index.tsx`.
