# Textarea

Use Textarea inside forms where native input behavior, labeling, validation, and token-driven styling need to stay aligned.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Textarea } from "@stylextras/ui/textarea";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>

</Suspense>

```tsx
import { Textarea } from "@stylextras/ui/textarea";

<Textarea label="Notes" rows={4} />;
```

## Props

The public prop types are exported from `@stylextras/ui/textarea`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type TextareaProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
  textareaSx?: StyleXStyles;
  labelSx?: StyleXStyles;
  label?: ReactNode;
  size?: TextareaSize;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { Textarea } from "@stylextras/ui/textarea";

<Textarea label="Label" sx={styles.emphasis} />;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { Textarea } from "@stylextras/ui/textarea";

function TextareaExample() {
  return (
    <>
      <div>
        <Textarea
          label="Summary"
          rows={4}
          defaultValue="Short multiline content."
        />
        <Textarea
          label="Notes"
          rows={6}
          defaultValue="Longer multiline content helps show the rhythm, spacing, and line height of the component without extra layout noise."
        />
      </div>
    </>
  );
}
```

## Accessibility notes

Every interactive form control should have a label or accessible name. Preserve native attributes such as required, disabled, min, max, and aria-describedby when they apply.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/textarea/index.tsx`.
