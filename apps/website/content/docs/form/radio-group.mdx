# Radio Group

Use Radio Group inside forms where native input behavior, labeling, validation, and token-driven styling need to stay aligned.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { RadioGroup } from "@stylextras/ui/radio-group";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { RadioGroup } from "@stylextras/ui/radio-group";

<RadioGroup>Example</RadioGroup>;
```

## Props

The public prop types are exported from `@stylextras/ui/radio-group`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type RadioGroupProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
  legendSx?: StyleXStyles;
  optionSx?: StyleXStyles;
  legend?: ReactNode;
  name?: string;
  options?: RadioOption[];
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { RadioGroup } from "@stylextras/ui/radio-group";

<RadioGroup sx={styles.emphasis}>Example</RadioGroup>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { RadioGroup } from "@stylextras/ui/radio-group";

function RadioGroupExample() {
  return (
    <>
      <RadioGroup
        legend="Where should this release go?"
        name="publish-target"
        options={["Preview only", "Staging", "Production"]}
      />
    </>
  );
}
```

## Accessibility notes

Every interactive form control should have a label or accessible name. Preserve native attributes such as required, disabled, min, max, and aria-describedby when they apply.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/radio-group/index.tsx`.
