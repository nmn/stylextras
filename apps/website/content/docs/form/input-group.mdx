# Input Group

Use Input Group inside forms where native input behavior, labeling, validation, and token-driven styling need to stay aligned.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { InputGroup } from "@stylextras/ui/input-group";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { InputGroup } from "@stylextras/ui/input-group";

<InputGroup>Example</InputGroup>;
```

## Props

The public prop types are exported from `@stylextras/ui/input-group`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type InputGroupProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { InputGroup } from "@stylextras/ui/input-group";

<InputGroup sx={styles.emphasis}>Example</InputGroup>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { InputGroup } from "@stylextras/ui/input-group";
import { TextField } from "@stylextras/ui/text-field";
import { Button } from "@stylextras/ui/button";

function InputGroupExample() {
  return (
    <>
      <InputGroup>
        <TextField label="Invite email" placeholder="teammate@example.com" />
        <Button>Invite</Button>
      </InputGroup>
    </>
  );
}
```

## Accessibility notes

Every interactive form control should have a label or accessible name. Preserve native attributes such as required, disabled, min, max, and aria-describedby when they apply.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/input-group/index.tsx`.
