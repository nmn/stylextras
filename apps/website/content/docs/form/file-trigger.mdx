# File Trigger

Use File Trigger inside forms where native input behavior, labeling, validation, and token-driven styling need to stay aligned.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { FileTrigger } from "@stylextras/ui/file-trigger";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { FileTrigger } from "@stylextras/ui/file-trigger";

<FileTrigger>Example</FileTrigger>;
```

## Props

The public prop types are exported from `@stylextras/ui/file-trigger`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type FileTriggerProps = Omit<
  BaseProps,
  "className" | "style" | "type"
> & {
  sx?: StyleXStyles;
  label?: ReactNode;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { FileTrigger } from "@stylextras/ui/file-trigger";

<FileTrigger label="Label" sx={styles.emphasis} />;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { FileTrigger } from "@stylextras/ui/file-trigger";

function FileTriggerExample() {
  return (
    <>
      <div>
        <FileTrigger label="Choose image" accept="image/*" />
        <FileTrigger label="Choose JSON" accept="application/json" />
      </div>
    </>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the popup body can be prefetched and rendered only when needed.

```tsx
import { FileTrigger } from "@stylextras/ui/file-trigger";

function DetailsContent() {
  return (
    <undefined>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </undefined>
  );
}

<FileTrigger content={() => Promise.resolve(DetailsContent)}>
  Open File Trigger
</FileTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import {} from "@stylextras/ui/file-trigger";

<undefined>
  Content can still be rendered directly when another controller owns opening.
</undefined>;
```

## Accessibility notes

Every interactive form control should have a label or accessible name. Preserve native attributes such as required, disabled, min, max, and aria-describedby when they apply.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/file-trigger/index.tsx`.
