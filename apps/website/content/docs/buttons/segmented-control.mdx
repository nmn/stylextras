# Segmented Control

Use Segmented Control when an interface needs a clear, keyboard-accessible control for taking action. The component keeps native button behavior and moves visual decisions into token variables.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { SegmentedControl } from "@stylextras/ui/segmented-control";
```

## Basic usage

Start with a compact set of short, mutually exclusive choices. The component uses native radio inputs, so a visible `legend` gives the group an accessible name.

<Suspense fallback={null}>
  <BasicSegmentedControlDemo />
</Suspense>

```tsx
import { SegmentedControl } from "@stylextras/ui/segmented-control";

<SegmentedControl
  defaultValue="preview"
  legend="View"
  options={[
    { label: "Preview", value: "preview" },
    { label: "Code", value: "code" },
    { label: "Tokens", value: "tokens" },
  ]}
/>;
```

## Dense choices

Use this form in toolbars, filters, and other compact control areas. Keep labels short so the control can wrap cleanly on narrow screens.

## Descriptive choices

Descriptions are useful when labels alone are ambiguous. Use them sparingly because each segment becomes taller and needs more horizontal space.

## Props

The public prop types are exported from `@stylextras/ui/segmented-control`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type SegmentedControlProps = Omit<BaseProps, "className" | "style"> & {
  legend?: ReactNode;
  name?: string;
  onValueChange?: (value: string) => void;
  options?: SegmentOption[];
  sx?: StyleXStyles;
  value?: string;
  defaultValue?: string;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { SegmentedControl } from "@stylextras/ui/segmented-control";

<SegmentedControl sx={styles.emphasis}>Example</SegmentedControl>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Controlled value

Control the value when the selected segment needs to update application state or drive another view.

```tsx
import { useState } from "react";
import { SegmentedControl } from "@stylextras/ui/segmented-control";

function SegmentedControlExample() {
  const [view, setView] = useState("preview");

  return (
    <SegmentedControl
      value={view}
      onValueChange={setView}
      legend="View"
      options={[
        { label: "Preview", value: "preview" },
        { label: "Code", value: "code" },
        { label: "Tokens", value: "tokens" },
      ]}
    />
  );
}
```

## Accessibility notes

Use a `legend` for every control so the native radio group has a clear accessible name. Keep segment labels concise, and use descriptions only when the extra text changes the user's decision.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/segmented-control/index.tsx`.
