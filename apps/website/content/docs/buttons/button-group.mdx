# Button Group

Use Button Group when related buttons should read as one control cluster. Use the action-row variant for dialog footers and confirmation rows where the primary and secondary actions should be equal width and aligned to the end.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { ButtonGroup, ButtonGroupActions } from "@stylextras/ui/button-group";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { Button } from "@stylextras/ui/button";
import { ButtonGroup } from "@stylextras/ui/button-group";

<ButtonGroup>
  <Button>Approve</Button>
  <Button variant="secondary">Comment</Button>
  <Button variant="outline">Request changes</Button>
</ButtonGroup>;
```

## Action row

Use `ButtonGroupActions` when the buttons should keep a gap instead of sharing borders. The primary and secondary buttons are equal width and right-aligned; a leading button can be added for a tertiary action or icon-only affordance.

```tsx
import { Button } from "@stylextras/ui/button";
import { ButtonGroupActions } from "@stylextras/ui/button-group";
import { IconButton } from "@stylextras/ui/icon-button";

<ButtonGroupActions
  leading={<IconButton label="More actions">+</IconButton>}
  secondary={<Button variant="secondary">Cancel</Button>}
  primary={<Button>Save changes</Button>}
/>;
```

## Props

The public prop types are exported from `@stylextras/ui/button-group`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type ButtonGroupProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
};

export type ButtonGroupActionsProps = Omit<
  BaseProps,
  "children" | "className" | "style"
> & {
  leading?: ReactNode;
  primary: ReactNode;
  secondary: ReactNode;
  sx?: StyleXStyles;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { Button } from "@stylextras/ui/button";
import { ButtonGroup } from "@stylextras/ui/button-group";

<ButtonGroup sx={styles.emphasis}>
  <Button>One</Button>
  <Button variant="secondary">Two</Button>
</ButtonGroup>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { Button } from "@stylextras/ui/button";
import { ButtonGroup, ButtonGroupActions } from "@stylextras/ui/button-group";
import { IconButton } from "@stylextras/ui/icon-button";

function ButtonGroupExample() {
  return (
    <div>
      <ButtonGroup>
        <Button>Approve</Button>
        <Button variant="secondary">Comment</Button>
        <Button variant="outline">Request changes</Button>
      </ButtonGroup>

      <ButtonGroupActions
        leading={<IconButton label="More actions">+</IconButton>}
        secondary={<Button variant="secondary">Cancel</Button>}
        primary={<Button>Save changes</Button>}
      />
    </div>
  );
}
```

## Grouped composition

Group related items when the collection should scan as a single unit.

```tsx
import { ButtonGroup } from "@stylextras/ui/button-group";

<ButtonGroup>
  <Button>One</Button>
  <Button variant="secondary">Two</Button>
</ButtonGroup>;
```

## Dialog actions

Use action rows for dialogs and drawers because they preserve a visible gap between the buttons and keep the primary action easy to scan.

```tsx
import { Button } from "@stylextras/ui/button";
import { ButtonGroupActions } from "@stylextras/ui/button-group";

<form method="dialog">
  <ButtonGroupActions
    secondary={
      <Button type="submit" variant="secondary">
        Cancel
      </Button>
    }
    primary={<Button type="submit">Save</Button>}
  />
</form>;
```

## Accessibility notes

Prefer explicit visible text for actions. For icon-only controls, use the component props that require an accessible label instead of relying on the icon alone.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
