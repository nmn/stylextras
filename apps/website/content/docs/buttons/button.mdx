# Button

Use Button when an interface needs a clear, keyboard-accessible control for taking action. The component keeps native button behavior and moves visual decisions into token variables.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Button } from "@stylextras/ui/button";
```

## Basic usage

Start with a semantic button and choose a variant that matches the action's priority. Group the visual variants in one example because they are easiest to compare side by side.

## Props

The public prop types are exported from `@stylextras/ui/button`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type ButtonVariant =
  | "primary"
  | "secondary"
  | "tertiary"
  | "outline"
  | "ghost"
  | "danger"
  | "critical"
  | "critical-outline";

export type ButtonSize = "sm" | "md" | "lg";

export type ButtonProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
  variant?: ButtonVariant;
  size?: ButtonSize;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { Button } from "@stylextras/ui/button";

<Button sx={styles.emphasis}>Example</Button>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Variants

Use variants when the same component needs different visual priority or intent.

```tsx

<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="tertiary">Tertiary</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="danger">Danger</Button>
<Button variant="critical">Critical</Button>
<Button variant="critical-outline">Critical outline</Button>
```

## Sizes

Use sizes to fit the control into dense or spacious layouts without changing semantics.

```tsx
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>
```

## Accessibility notes

Prefer explicit visible text for actions. For icon-only controls, use the component props that require an accessible label instead of relying on the icon alone.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/button/index.tsx`.
