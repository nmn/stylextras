# Tooltip

Use Tooltip for portal-free popup interactions built from trigger and content components. These docs show lazy content loading and direct content composition.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import {
  TooltipContent,
  type TooltipContentProps,
  TooltipTrigger,
} from "@stylextras/ui/tooltip";
```

## Basic usage

Start here when wiring the component into an interface. Tooltips open on hover and keyboard focus, not on click.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import {
  TooltipContent,
  type TooltipContentProps,
  TooltipTrigger,
} from "@stylextras/ui/tooltip";

function TooltipExampleContent(props: TooltipContentProps) {
  return <TooltipContent {...props}>Details</TooltipContent>;
}

<TooltipTrigger content={() => Promise.resolve(TooltipExampleContent)}>
  Export token snapshot
</TooltipTrigger>;
```

## Props

The public prop types are exported from `@stylextras/ui/tooltip`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type TooltipProps = Omit<
  BaseProps,
  "className" | "style" | "popover"
> & {
  placement?: TooltipPlacement;
  sx?: StyleXStyles;
};

export type TooltipContentProps = TooltipProps;

export type TooltipTriggerProps = Omit<
  ButtonProps,
  "className" | "content" | "style"
> & {
  content: LazyComponentLoader<TooltipContentProps>;
  contentProps?: Omit<TooltipContentProps, "ref">;
  fallback?: React.ReactNode;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import {
  TooltipContent,
  type TooltipContentProps,
  TooltipTrigger,
} from "@stylextras/ui/tooltip";

<TooltipContent sx={styles.emphasis}>Example</TooltipContent>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows hover and focus disclosure around a specific control.

```tsx
import {
  TooltipContent,
  type TooltipContentProps,
  TooltipTrigger,
} from "@stylextras/ui/tooltip";

function TooltipExampleContent(props: TooltipContentProps) {
  return (
    <TooltipContent {...props}>
      Creates a static JSON snapshot of the current theme.
    </TooltipContent>
  );
}

function TooltipExample() {
  return (
    <div>
      <TooltipTrigger content={() => Promise.resolve(TooltipExampleContent)}>
        Export token snapshot
      </TooltipTrigger>
    </div>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the tooltip body can be prefetched on hover or focus and rendered only when needed.

```tsx
import {
  TooltipTrigger,
  TooltipContent,
  type TooltipContentProps,
} from "@stylextras/ui/tooltip";

function DetailsContent(props: TooltipContentProps) {
  return (
    <TooltipContent {...props}>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </TooltipContent>
  );
}

<TooltipTrigger content={() => Promise.resolve(DetailsContent)}>
  Export token snapshot
</TooltipTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import { TooltipContent } from "@stylextras/ui/tooltip";

<TooltipContent>
  Content can still be rendered directly when another controller owns opening.
</TooltipContent>;
```

## Accessibility notes

Keep trigger text specific, label the popup content when it has a title, and prefer the platform dialog or popover behavior before adding custom focus management.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/tooltip/index.tsx`.
