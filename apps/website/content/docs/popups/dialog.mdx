# Dialog

Use Dialog for portal-free popup interactions built from trigger and content components. These docs show lazy content loading and direct content composition.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import {
  DialogContent,
  DialogTrigger,
  DialogProgrammatic,
} from "@stylextras/ui/dialog";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import {
  DialogContent,
  DialogTrigger,
  DialogProgrammatic,
} from "@stylextras/ui/dialog";

function DialogExampleContent(props: DialogContentProps) {
  return <DialogContent {...props}>Details</DialogContent>;
}

<DialogTrigger content={() => Promise.resolve(DialogExampleContent)}>
  Open Dialog
</DialogTrigger>;
```

## Props

The public prop types are exported from `@stylextras/ui/dialog`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type DialogProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
  size?: DialogSize;
};

export type DialogContentProps = DialogProps;

export type DialogTriggerProps = Omit<
  ButtonProps,
  "className" | "content" | "style"
> & {
  content: LazyComponentLoader<DialogContentProps>;
  contentProps?: Omit<DialogContentProps, "ref">;
  fallback?: React.ReactNode;
};

export type DialogProgrammaticProps<
  Props extends ComponentPropsWithRef<"dialog"> = DialogContentProps,
> = {
  content: LazyComponentLoader<Props>;
  contentProps?: Omit<Props, "open" | "ref">;
  defaultOpen?: boolean;
  fallback?: React.ReactNode;
  onOpenChange?: (open: boolean) => void;
  open?: boolean;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import {
  DialogContent,
  DialogTrigger,
  DialogProgrammatic,
} from "@stylextras/ui/dialog";

<DialogContent sx={styles.emphasis}>Example</DialogContent>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { Button } from "@stylextras/ui/button";
import { ButtonGroup } from "@stylextras/ui/button-group";
import { TextField } from "@stylextras/ui/text-field";
import { Typography } from "@stylextras/ui/typography";
import {
  DialogContent,
  type DialogContentProps,
  DialogTrigger,
} from "@stylextras/ui/dialog";

function DialogExampleContent(props: DialogContentProps) {
  return (
    <DialogContent {...props}>
      <div>
        <Typography as="h3" scale="title">
          Rename component
        </Typography>
        <TextField label="Display name" defaultValue="Segmented Control" />
        <form method="dialog">
          <ButtonGroup>
            <Button type="submit" variant="secondary">
              Cancel
            </Button>
            <Button type="submit">Save</Button>
          </ButtonGroup>
        </form>
      </div>
    </DialogContent>
  );
}

function DialogExample() {
  return (
    <DialogTrigger content={() => Promise.resolve(DialogExampleContent)}>
      Open dialog
    </DialogTrigger>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the popup body can be prefetched and rendered only when needed.

```tsx
import {
  DialogTrigger,
  DialogContent,
  type DialogContentProps,
} from "@stylextras/ui/dialog";

function DetailsContent(props: DialogContentProps) {
  return (
    <DialogContent {...props}>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </DialogContent>
  );
}

<DialogTrigger content={() => Promise.resolve(DetailsContent)}>
  Open Dialog
</DialogTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import { DialogContent } from "@stylextras/ui/dialog";

<DialogContent>
  Content can still be rendered directly when another controller owns opening.
</DialogContent>;
```

## Programmatic control

Use the programmatic component when open state belongs to application state rather than a trigger.

```tsx
import {
  DialogProgrammatic,
  DialogContent,
  type DialogContentProps,
} from "@stylextras/ui/dialog";

function ProgrammaticExample({ open, onOpenChange }) {
  return (
    <DialogProgrammatic
      open={open}
      onOpenChange={onOpenChange}
      content={() =>
        Promise.resolve((props: DialogContentProps) => (
          <DialogContent {...props}>Controlled content</DialogContent>
        ))
      }
    />
  );
}
```

## Accessibility notes

Keep trigger text specific, label the popup content when it has a title, and prefer the platform dialog or popover behavior before adding custom focus management.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/dialog/index.tsx`.
