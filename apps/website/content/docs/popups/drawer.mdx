# Drawer

Use Drawer for portal-free popup interactions built from trigger and content components. These docs show lazy content loading and direct content composition.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import {
  DrawerContent,
  DrawerTrigger,
  DrawerProgrammatic,
} from "@stylextras/ui/drawer";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import {
  DrawerContent,
  DrawerTrigger,
  DrawerProgrammatic,
} from "@stylextras/ui/drawer";

function DrawerExampleContent(props: DrawerContentProps) {
  return <DrawerContent {...props}>Details</DrawerContent>;
}

<DrawerTrigger content={() => Promise.resolve(DrawerExampleContent)}>
  Open Drawer
</DrawerTrigger>;
```

## Props

The public prop types are exported from `@stylextras/ui/drawer`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type DrawerProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
  side?: DrawerSide;
};

export type DrawerContentProps = DrawerProps;

export type DrawerTriggerProps = Omit<
  ButtonProps,
  "className" | "content" | "style"
> & {
  content: LazyComponentLoader<DrawerContentProps>;
  contentProps?: Omit<DrawerContentProps, "ref">;
  fallback?: React.ReactNode;
};

export type DrawerProgrammaticProps =
  DialogProgrammaticProps<DrawerContentProps>;
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import {
  DrawerContent,
  DrawerTrigger,
  DrawerProgrammatic,
} from "@stylextras/ui/drawer";

<DrawerContent sx={styles.emphasis}>Example</DrawerContent>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { Button } from "@stylextras/ui/button";
import { ButtonGroup } from "@stylextras/ui/button-group";
import { Typography } from "@stylextras/ui/typography";
import {
  DrawerContent,
  type DrawerContentProps,
  DrawerTrigger,
} from "@stylextras/ui/drawer";

function DrawerExampleContent(props: DrawerContentProps) {
  return (
    <DrawerContent {...props}>
      <div>
        <Typography as="h3" scale="title">
          Page settings
        </Typography>
        <span>
          Use a drawer for nearby controls without leaving the current page.
        </span>
        <form method="dialog">
          <ButtonGroup>
            <Button type="submit" variant="secondary">
              Close
            </Button>
            <Button type="submit">Save changes</Button>
          </ButtonGroup>
        </form>
      </div>
    </DrawerContent>
  );
}

function DrawerExample() {
  return (
    <DrawerTrigger content={() => Promise.resolve(DrawerExampleContent)}>
      Open drawer
    </DrawerTrigger>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the popup body can be prefetched and rendered only when needed.

```tsx
import {
  DrawerTrigger,
  DrawerContent,
  type DrawerContentProps,
} from "@stylextras/ui/drawer";

function DetailsContent(props: DrawerContentProps) {
  return (
    <DrawerContent {...props}>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </DrawerContent>
  );
}

<DrawerTrigger content={() => Promise.resolve(DetailsContent)}>
  Open Drawer
</DrawerTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import { DrawerContent } from "@stylextras/ui/drawer";

<DrawerContent>
  Content can still be rendered directly when another controller owns opening.
</DrawerContent>;
```

## Programmatic control

Use the programmatic component when open state belongs to application state rather than a trigger.

```tsx
import {
  DrawerProgrammatic,
  DrawerContent,
  type DrawerContentProps,
} from "@stylextras/ui/drawer";

function ProgrammaticExample({ open, onOpenChange }) {
  return (
    <DrawerProgrammatic
      open={open}
      onOpenChange={onOpenChange}
      content={() =>
        Promise.resolve((props: DrawerContentProps) => (
          <DrawerContent {...props}>Controlled content</DrawerContent>
        ))
      }
    />
  );
}
```

## Accessibility notes

Keep trigger text specific, label the popup content when it has a title, and prefer the platform dialog or popover behavior before adding custom focus management.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/drawer/index.tsx`.
