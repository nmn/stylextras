# Hover Card

Use Hover Card for portal-free popup interactions built from trigger and content components. These docs show lazy content loading and direct content composition.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";
```

## Basic usage

Start here when wiring the component into an interface. Hover cards open on hover and keyboard focus, not on click.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";

function HoverCardExampleContent(props: HoverCardContentProps) {
  return <HoverCardContent {...props}>Details</HoverCardContent>;
}

<HoverCardTrigger content={() => Promise.resolve(HoverCardExampleContent)}>
  Open preview
</HoverCardTrigger>;
```

## Props

The public prop types are exported from `@stylextras/ui/hover-card`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type HoverCardProps = Omit<
  BaseProps,
  "className" | "style" | "popover"
> & {
  placement?: HoverCardPlacement;
  sx?: StyleXStyles;
};

export type HoverCardContentProps = HoverCardProps;

export type HoverCardTriggerProps = Omit<
  ButtonProps,
  "className" | "content" | "style"
> & {
  content: LazyComponentLoader<HoverCardContentProps>;
  contentProps?: Omit<HoverCardContentProps, "ref">;
  fallback?: React.ReactNode;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";

<HoverCardContent sx={styles.emphasis}>Example</HoverCardContent>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows preview content that appears while the trigger is hovered or focused.

```tsx
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";

function HoverCardExampleContent(props: HoverCardContentProps) {
  return (
    <HoverCardContent {...props}>
      <div>
        <strong>Package preview</strong>
        <span>Lightweight summary content belongs here.</span>
      </div>
    </HoverCardContent>
  );
}

function HoverCardExample() {
  return (
    <HoverCardTrigger content={() => Promise.resolve(HoverCardExampleContent)}>
      Open preview
    </HoverCardTrigger>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the hover card body can be prefetched on hover or focus and rendered only when needed.

```tsx
import {
  HoverCardTrigger,
  HoverCardContent,
  type HoverCardContentProps,
} from "@stylextras/ui/hover-card";

function DetailsContent(props: HoverCardContentProps) {
  return (
    <HoverCardContent {...props}>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </HoverCardContent>
  );
}

<HoverCardTrigger content={() => Promise.resolve(DetailsContent)}>
  Open preview
</HoverCardTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import { HoverCardContent } from "@stylextras/ui/hover-card";

<HoverCardContent>
  Content can still be rendered directly when another controller owns opening.
</HoverCardContent>;
```

## Accessibility notes

Keep trigger text specific, label the popup content when it has a title, and prefer the platform dialog or popover behavior before adding custom focus management.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/hover-card/index.tsx`.
