# Hover Card

Use Hover Card for portal-free preview interactions built from trigger and content components. The trigger remains a normal button or link: clicking it keeps its native behavior. Hover and focus first open a tooltip-style preview, and ArrowDown or hovering the preview upgrades it to a modal dialog.

The tooltip preview hides the rich card body from assistive technology and exposes a helper prompt instead. Dialog mode uses the same visual surface, exposes the full content, and moves focus into the dialog.

## Import

```tsx
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";
```

## Basic usage

Start here when wiring the component into an interface. Hover cards open on hover and keyboard focus, not on click. Press Escape to dismiss the tooltip preview. Press ArrowDown to open the full dialog.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";

function HoverCardExampleContent(props: HoverCardContentProps) {
  return (
    <HoverCardContent {...props}>
      <strong>Package preview</strong>
      <p>Rich content is available after the preview upgrades to a dialog.</p>
      <button type="button">Focusable action</button>
    </HoverCardContent>
  );
}

<HoverCardTrigger content={() => Promise.resolve(HoverCardExampleContent)}>
  Open preview
</HoverCardTrigger>;
```

## Link triggers

Triggers can also be anchors. The click action is not intercepted, so the link still navigates when activated.

```tsx
<HoverCardTrigger
  as="a"
  content={() => Promise.resolve(HoverCardExampleContent)}
  href="/docs"
>
  Documentation
</HoverCardTrigger>
```

## Tooltip and dialog behavior

The same content component renders in two modes. Tooltip mode wraps the visual content in an inert, screen-reader-hidden preview and adds the helper text. Dialog mode removes the helper text and exposes the full content inside a native modal dialog.

```tsx
function ProfilePreview(props: HoverCardContentProps) {
  return (
    <HoverCardContent {...props} placement="right">
      <h2>Robin West</h2>
      <p>Design systems engineer focused on platform-native UI.</p>
      <button type="button">View profile</button>
    </HoverCardContent>
  );
}
```

## Props

The public prop types are exported from `@stylextras/ui/hover-card`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type HoverCardProps = Omit<
  DivProps & DialogProps,
  "className" | "style" | "popover" | "ref"
> & {
  anchor?: string;
  helperText?: ReactNode;
  mode?: "tooltip" | "dialog";
  placement?: HoverCardPlacement;
  ref?: Ref<HTMLDivElement | HTMLDialogElement>;
  sx?: StyleXStyles;
};

export type HoverCardContentProps = HoverCardProps;

export type HoverCardTriggerButtonProps = Omit<
  ButtonProps,
  "className" | "content" | "style"
> & {
  as?: "button";
  content: LazyComponentLoader<HoverCardContentProps>;
  contentProps?: Omit<HoverCardContentProps, "mode" | "ref">;
  fallback?: ReactNode;
  helperText?: ReactNode;
};

export type HoverCardTriggerLinkProps = Omit<
  ComponentPropsWithRef<"a">,
  "className" | "content" | "style"
> &
  HoverCardTriggerButtonProps & {
    as: "a";
  };
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";

<HoverCardContent sx={styles.emphasis}>Example</HoverCardContent>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows rich preview content that starts inert in tooltip mode and becomes interactive in dialog mode.

```tsx
import {
  HoverCardContent,
  type HoverCardContentProps,
  HoverCardTrigger,
} from "@stylextras/ui/hover-card";

function HoverCardExampleContent(props: HoverCardContentProps) {
  return (
    <HoverCardContent {...props}>
      <div>
        <strong>Package preview</strong>
        <span>Press ArrowDown or hover the preview to inspect more.</span>
        <button type="button">Focusable action</button>
      </div>
    </HoverCardContent>
  );
}

function HoverCardExample() {
  return (
    <HoverCardTrigger content={() => Promise.resolve(HoverCardExampleContent)}>
      Open preview
    </HoverCardTrigger>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the hover card body can be prefetched on hover or focus and rendered only when needed. The trigger preloads without changing click behavior.

```tsx
import {
  HoverCardTrigger,
  HoverCardContent,
  type HoverCardContentProps,
} from "@stylextras/ui/hover-card";

function DetailsContent(props: HoverCardContentProps) {
  return (
    <HoverCardContent {...props}>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </HoverCardContent>
  );
}

<HoverCardTrigger content={() => Promise.resolve(DetailsContent)}>
  Open preview
</HoverCardTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import { HoverCardContent } from "@stylextras/ui/hover-card";

<HoverCardContent>
  Content can still be rendered directly when another controller owns opening.
</HoverCardContent>;
```

## Accessibility notes

- Triggers remain normal buttons or links.
- Tooltip mode uses `role="tooltip"` and only exposes the helper prompt to assistive technology.
- Rich preview content is inert in tooltip mode.
- ArrowDown upgrades the tooltip into a native modal dialog.
- Hovering the tooltip preview also upgrades it into a dialog.
- Escape dismisses the tooltip or dialog.
- Dialog dismissal returns focus to the trigger.

- Styles use token variables from `@stylextras/ui/tokens/*`.
