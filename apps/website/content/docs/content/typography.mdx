# Typography

Use Typography to shape reusable content blocks while keeping typography, spacing, and surface styling token-driven.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Typography } from "@stylextras/ui/typography";
```

## Basic usage

Choose a scale for the visual role of the text, then choose `as` for the semantic element that belongs in the surrounding document.

<Suspense fallback={null}>
  <TypographyScalesDemo />
</Suspense>

```tsx
import { Typography } from "@stylextras/ui/typography";

<Typography scale="display" as="h1">Display</Typography>
<Typography scale="title" as="h2">Title</Typography>
<Typography scale="body" as="p">Body copy</Typography>
<Typography scale="label" as="span">Label</Typography>
```

## Semantics and tones

Use `as` to preserve the page outline and `tone` to lower emphasis without changing markup.

<Suspense fallback={null}>
  <TypographySemanticsDemo />
</Suspense>

```tsx
<Typography as="h1" scale="display">Heading 1</Typography>
<Typography as="h2" scale="title">Heading 2</Typography>
<Typography as="p" tone="muted">Muted paragraph</Typography>
<Typography as="p" tone="brand">Brand paragraph</Typography>
```

## Props

The public prop types are exported from `@stylextras/ui/typography`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type TypographyProps = Omit<BaseProps, "className" | "style"> & {
  as?: ElementType;
  sx?: StyleXStyles;
  tone?: TypographyTone;
  scale?: TypographyScale;
  mono?: boolean;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { Typography } from "@stylextras/ui/typography";

<Typography sx={styles.emphasis}>Example</Typography>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Accessibility notes

Choose semantic children and heading levels based on the page structure around the component.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/typography/index.tsx`.
