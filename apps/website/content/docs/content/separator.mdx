# Separator

Use Separator to shape reusable content blocks while keeping typography, spacing, and surface styling token-driven.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Separator } from "@stylextras/ui/separator";
```

## Basic usage

Use the default horizontal orientation between stacked pieces of content. Add an accessible label only when the separator carries meaning.

<Suspense fallback={null}>
  <HorizontalSeparatorDemo />
</Suspense>

```tsx
import { Separator } from "@stylextras/ui/separator";

<span>Above</span>
<Separator />
<span>Below</span>
```

## Vertical orientation

Use a vertical separator inside inline control groups or metadata rows where the visual divider follows the reading order.

  <VerticalSeparatorDemo />
</Suspense>

```tsx
<span>Left</span>
<Separator orientation="vertical" />
<span>Right</span>
```

## Props

The public prop types are exported from `@stylextras/ui/separator`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type SeparatorProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
  orientation?: SeparatorOrientation;
  emphasis?: SeparatorEmphasis;
  decorative?: boolean;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { Separator } from "@stylextras/ui/separator";

<Separator aria-label="Example" sx={styles.emphasis} />;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Accessibility notes

Choose semantic children and heading levels based on the page structure around the component.

- Styles use token variables from `@stylextras/ui/tokens/*`.
