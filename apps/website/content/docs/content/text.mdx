# Text

Use Text to shape reusable content blocks while keeping typography, spacing, and surface styling token-driven.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Text } from "@stylextras/ui/text";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { Text } from "@stylextras/ui/text";

<Text>Example</Text>;
```

## Props

The public prop types are exported from `@stylextras/ui/text`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type TextProps = Omit<TypographyProps, "scale">;
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { Text } from "@stylextras/ui/text";

function TextExample() {
  return (
    <>
      <div>
        <Text>Use short, specific sentences around interactive controls.</Text>
        <Text tone="muted">
          Muted text is useful for supporting descriptions and secondary notes.
        </Text>
        <Text mono>
          Monospace body text can help with code-like labels and values.
        </Text>
      </div>
    </>
  );
}
```

## Tone variants

This pattern shows another common way to compose the component.

```tsx
import { Text } from "@stylextras/ui/text";

<Text>Default body text</Text>;
<Text tone="muted">Muted supporting text</Text>;
```
