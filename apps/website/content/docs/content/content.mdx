# Content

Use Content to shape reusable content blocks while keeping typography, spacing, and surface styling token-driven.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Content } from "@stylextras/ui/content";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { Content } from "@stylextras/ui/content";

<Content>Example</Content>;
```

## Props

The public prop types are exported from `@stylextras/ui/content`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type ContentProps = Omit<BaseProps, "className" | "style"> & {
  as?: ContentElement;
  sx?: StyleXStyles;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { Content } from "@stylextras/ui/content";

<Content sx={styles.emphasis}>Example</Content>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { Content } from "@stylextras/ui/content";
import { Typography } from "@stylextras/ui/typography";
import { Text } from "@stylextras/ui/text";

function ContentExample() {
  return (
    <>
      <Content>
        <Typography as="h2" scale="title">
          Content wrapper
        </Typography>
        <Text>
          Use Content to constrain long-form copy into a readable column width.
        </Text>
        <Text>
          It works best for guides, articles, and documentation sections where
          line length matters more than interactive chrome.
        </Text>
      </Content>
    </>
  );
}
```

Choose semantic children and heading levels based on the page structure around the component.
