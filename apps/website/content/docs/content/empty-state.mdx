# Empty State

Use Empty State to shape reusable content blocks while keeping typography, spacing, and surface styling token-driven.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

Search aliases: empty, placeholder state, no results, no data.

## Import

```tsx
import { EmptyState } from "@stylextras/ui/empty-state";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { EmptyState } from "@stylextras/ui/empty-state";

<EmptyState>Example</EmptyState>;
```

## Props

The public prop types are exported from `@stylextras/ui/empty-state`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type EmptyStateProps = EmptyProps;
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { EmptyState } from "@stylextras/ui/empty-state";

function EmptyStateExample() {
  return (
    <>
      <EmptyState>
        <div>
          <strong>No items yet</strong>
          <span>Create the first item to populate this view.</span>
        </div>
      </EmptyState>
    </>
  );
}
```

## With a recovery action

This pattern shows another common way to compose the component.

```tsx
import { EmptyState } from "@stylextras/ui/empty-state";
import { Button } from "@stylextras/ui/button";

<EmptyState>
  <strong>No projects yet</strong>
  <span>Create a project to populate this workspace.</span>
  <Button>Create project</Button>
</EmptyState>;
```
