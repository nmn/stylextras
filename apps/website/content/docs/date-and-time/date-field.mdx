# Date Field

Use Date Field for date or time entry flows that can lean on native form controls and browser validation.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { DateField } from "@stylextras/ui/date-field";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { DateField } from "@stylextras/ui/date-field";

<DateField label="Start date" defaultValue="2026-04-10" />;
```

## Props

The public prop types are exported from `@stylextras/ui/date-field`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type DateFieldProps = Omit<BaseProps, "className" | "style" | "type"> & {
  sx?: StyleXStyles;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { DateField } from "@stylextras/ui/date-field";

<DateField sx={styles.emphasis}>Example</DateField>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { DateField } from "@stylextras/ui/date-field";

function DateFieldExample() {
  return (
    <>
      <div>
        <DateField aria-label="Start date" defaultValue="2026-04-10" />
        <DateField aria-label="End date" defaultValue="2026-04-18" />
      </div>
    </>
  );
}
```
