# Menu

Use Menu to present related items, actions, or records while keeping structure and styling separate from application state.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Menu, MenuContent, MenuTrigger } from "@stylextras/ui/menu";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { Menu, MenuContent, MenuTrigger } from "@stylextras/ui/menu";

<Menu>Example</Menu>;
```

## Props

The public prop types are exported from `@stylextras/ui/menu`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type MenuProps = DropdownMenuProps;

export type MenuContentProps = DropdownMenuContentProps;

export type MenuTriggerProps = DropdownMenuTriggerProps;
```

## Complete example

This example shows item activation, round quick actions, and a submenu trigger inside the menu.

```tsx
import {
  MenuButton,
  MenuContent as MenuItems,
  MenuIconButton,
  MenuIconButtonGroup,
  MenuItem,
  MenuItemList,
  MenuSection,
  MenuSectionTitle,
} from "@stylextras/ui/menu/menu-content";
import {
  MenuContent,
  type MenuContentProps,
  MenuTrigger,
} from "@stylextras/ui/menu";

function MoveSubmenuContent(props: MenuContentProps) {
  return (
    <MenuContent {...props} placement="right">
      <MenuItems>
        <MenuSection>
          <MenuSectionTitle>Move to</MenuSectionTitle>
          <MenuItem icon="P" type="button">
            Planning
          </MenuItem>
          <MenuItem icon="R" type="button">
            Research
          </MenuItem>
          <MenuItem icon="S" type="button">
            Shipped
          </MenuItem>
        </MenuSection>
      </MenuItems>
    </MenuContent>
  );
}

function MenuExampleContent(props: MenuContentProps) {
  return (
    <MenuContent {...props}>
      <MenuItems>
        <MenuSection>
          <MenuButton endIcon="↗" icon="★" type="button">
            Open project
          </MenuButton>
        </MenuSection>

        <MenuSection>
          <MenuSectionTitle>Quick actions</MenuSectionTitle>
          <MenuIconButtonGroup>
            <MenuIconButton label="Pin item" size="sm" type="button">
              📌
            </MenuIconButton>
            <MenuIconButton label="Share item" size="sm" type="button">
              ↗
            </MenuIconButton>
            <MenuIconButton label="Delete item" size="sm" type="button">
              ✕
            </MenuIconButton>
          </MenuIconButtonGroup>
        </MenuSection>

        <MenuSection>
          <MenuSectionTitle>More</MenuSectionTitle>
          <MenuItemList>
            <MenuItem
              description="Edit the current page title"
              icon="✎"
              type="button"
            >
              Rename
            </MenuItem>
            <MenuTrigger
              content={() => Promise.resolve(MoveSubmenuContent)}
              contentProps={{ placement: "right" }}
              variant="ghost"
            >
              Move to ›
            </MenuTrigger>
            <MenuItem description="Create a second copy" icon="⧉" type="button">
              Duplicate
            </MenuItem>
            <MenuItem
              description="Move it out of the active list"
              icon="🗄"
              type="button"
            >
              Archive
            </MenuItem>
          </MenuItemList>
        </MenuSection>
      </MenuItems>
    </MenuContent>
  );
}

function MenuExample() {
  return (
    <MenuTrigger
      content={() => Promise.resolve(MenuExampleContent)}
      variant="outline"
    >
      Actions
    </MenuTrigger>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the popup body can be prefetched and rendered only when needed.

```tsx
import {
  MenuTrigger,
  MenuContent,
  type MenuContentProps,
} from "@stylextras/ui/menu";

function DetailsContent(props: MenuContentProps) {
  return (
    <MenuContent {...props}>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </MenuContent>
  );
}

<MenuTrigger content={() => Promise.resolve(DetailsContent)}>
  Open Menu
</MenuTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import { MenuContent } from "@stylextras/ui/menu";

<MenuContent>
  Content can still be rendered directly when another controller owns opening.
</MenuContent>;
```

## Accessibility notes

Provide meaningful item text, headings, and native roles where the surrounding product owns richer selection or keyboard behavior.
