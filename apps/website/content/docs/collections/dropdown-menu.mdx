# Dropdown Menu

Use Dropdown Menu to present related items, actions, or records while keeping structure and styling separate from application state.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuTrigger,
} from "@stylextras/ui/dropdown-menu";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

## Static menu

Use the direct `DropdownMenu` composition when the menu content can be present in the document. The trigger uses native `popovertarget`, and the menu remains portal-free.

```tsx
import { DropdownMenu } from "@stylextras/ui/dropdown-menu";
import { MenuContent, MenuItem } from "@stylextras/ui/menu/menu-content";

<DropdownMenu label="Actions">
  <MenuContent>
    <MenuItem type="button">Rename</MenuItem>
    <MenuItem type="button">Duplicate</MenuItem>
  </MenuContent>
</DropdownMenu>;
```

## Props

The public prop types are exported from `@stylextras/ui/dropdown-menu`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type DropdownMenuProps = Omit<
  BaseProps,
  "className" | "style" | "popover"
> & {
  behavior?: DropdownMenuBehavior;
  label?: ReactNode;
  menuSx?: StyleXStyles;
  placement?: DropdownMenuPlacement;
  sx?: StyleXStyles;
  triggerSx?: StyleXStyles;
};

export type DropdownMenuContentProps = Omit<
  BaseProps,
  "className" | "style" | "popover"
> & {
  behavior?: DropdownMenuBehavior;
  placement?: DropdownMenuPlacement;
  sx?: StyleXStyles;
};

export type DropdownMenuTriggerProps = Omit<
  ButtonProps,
  "className" | "content" | "style"
> & {
  content: LazyComponentLoader<DropdownMenuContentProps>;
  contentProps?: Omit<DropdownMenuContentProps, "ref">;
  fallback?: React.ReactNode;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuTrigger,
} from "@stylextras/ui/dropdown-menu";

<DropdownMenu label="Label" sx={styles.emphasis} />;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { IconButton } from "@stylextras/ui/icon-button";
import {
  MenuButton,
  MenuContent,
  MenuIconButtonGroup,
  MenuItem,
  MenuItemList,
  MenuSection,
  MenuSectionTitle,
} from "@stylextras/ui/menu/menu-content";
import {
  DropdownMenuContent,
  type DropdownMenuContentProps,
  DropdownMenuTrigger,
} from "@stylextras/ui/dropdown-menu";

function DropdownMenuExampleContent(props: DropdownMenuContentProps) {
  return (
    <DropdownMenuContent {...props}>
      <MenuContent>
        <MenuSection>
          <MenuButton endIcon="↗" icon="★" type="button">
            Open project
          </MenuButton>
        </MenuSection>

        <MenuSection>
          <MenuSectionTitle>Quick actions</MenuSectionTitle>
          <MenuIconButtonGroup>
            <IconButton label="Pin item" size="sm" type="button">
              📌
            </IconButton>
            <IconButton label="Share item" size="sm" type="button">
              ↗
            </IconButton>
            <IconButton label="Delete item" size="sm" type="button">
              ✕
            </IconButton>
          </MenuIconButtonGroup>
        </MenuSection>

        <MenuSection>
          <MenuSectionTitle>More</MenuSectionTitle>
          <MenuItemList>
            <MenuItem
              description="Edit the current page title"
              icon="✎"
              type="button"
            >
              Rename
            </MenuItem>
            <MenuItem description="Create a second copy" icon="⧉" type="button">
              Duplicate
            </MenuItem>
            <MenuItem
              description="Move it out of the active list"
              icon="🗄"
              type="button"
            >
              Archive
            </MenuItem>
          </MenuItemList>
        </MenuSection>
      </MenuContent>
    </DropdownMenuContent>
  );
}

function DropdownMenuExample() {
  return (
    <DropdownMenuTrigger
      content={() => Promise.resolve(DropdownMenuExampleContent)}
      variant="outline"
    >
      Actions
    </DropdownMenuTrigger>
  );
}
```

## Lazy trigger and content

Pass a content loader to the trigger so the popup body can be prefetched and rendered only when needed.

```tsx
import {
  DropdownMenuTrigger,
  DropdownMenuContent,
  type DropdownMenuContentProps,
} from "@stylextras/ui/dropdown-menu";

function DetailsContent(props: DropdownMenuContentProps) {
  return (
    <DropdownMenuContent {...props}>
      <h2>Details</h2>
      <p>Render the content only after the trigger is activated.</p>
    </DropdownMenuContent>
  );
}

<DropdownMenuTrigger content={() => Promise.resolve(DetailsContent)}>
  Open Dropdown Menu
</DropdownMenuTrigger>;
```

## Content rendered directly

Render the content component directly when another controller owns the open state.

```tsx
import { DropdownMenuContent } from "@stylextras/ui/dropdown-menu";

<DropdownMenuContent>
  Content can still be rendered directly when another controller owns opening.
</DropdownMenuContent>;
```

## Accessibility notes

Provide meaningful item text, headings, and native roles where the surrounding product owns richer selection or keyboard behavior.
