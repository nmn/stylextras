# Disclosure Group

Use Disclosure Group for navigation patterns that need predictable structure, token-driven spacing, and clear focus behavior.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { DisclosureGroup } from "@stylextras/ui/disclosure-group";
```

## Basic usage

Use a regular group when multiple disclosures can stay open. This is the best default for FAQs and settings groups where users compare sections.

<Suspense fallback={null}>
  <DisclosureGroupBasicDemo />
</Suspense>

```tsx
import { Disclosure } from "@stylextras/ui/disclosure";
import { DisclosureGroup } from "@stylextras/ui/disclosure-group";

<DisclosureGroup>
  <Disclosure summary="What ships in the starter?">
    Tokens, components, and the docs site.
  </Disclosure>
  <Disclosure summary="How much is themeable?">
    Color, spacing, radius, and typography are themeable.
  </Disclosure>
</DisclosureGroup>;
```

## Exclusive accordion

Use `exclusive` when the group should behave like an accordion and keep only one section open at a time.

<Suspense fallback={null}>
  <DisclosureGroupExclusiveDemo />
</Suspense>

```tsx
<DisclosureGroup exclusive>
  <Disclosure summary="Design tokens">
    Keep the token layer small and derive the rest from core values.
  </Disclosure>
  <Disclosure summary="Native primitives">
    Prefer details, dialog, popover, and real form controls.
  </Disclosure>
</DisclosureGroup>
```

## Compact spacing

Use `variant="compact"` when the group still needs separate disclosure surfaces but with tighter spacing.

<Suspense fallback={null}>
  <DisclosureGroupCompactDemo />
</Suspense>

```tsx
<DisclosureGroup variant="compact">
  <Disclosure summary="Daily summary">
    New comments, status changes, and handoff notes.
  </Disclosure>
  <Disclosure summary="Open blockers">
    Tasks that need a decision before the next release.
  </Disclosure>
  <Disclosure summary="Follow-ups">
    People and projects that need another pass.
  </Disclosure>
</DisclosureGroup>
```

## Joined variant

Use `variant="joined"` when the disclosures should read as one compact accordion with no gaps between items.

```tsx
<DisclosureGroup exclusive variant="joined">

  </Disclosure>
  <Disclosure summary="Publishing">

```

## Props

The public prop types are exported from `@stylextras/ui/disclosure-group`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type DisclosureGroupProps = Omit<BaseProps, "className" | "style"> & {
  exclusive?: boolean;
  name?: string;
  variant?: DisclosureGroupVariant;
  sx?: StyleXStyles;
};

export type DisclosureGroupVariant = "spacious" | "compact" | "joined";
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { DisclosureGroup } from "@stylextras/ui/disclosure-group";

<DisclosureGroup sx={styles.emphasis}>Example</DisclosureGroup>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Accessibility notes

Use real links for navigation, mark the current location where appropriate, and keep focus order consistent with visual order.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/disclosure-group/index.tsx`.
