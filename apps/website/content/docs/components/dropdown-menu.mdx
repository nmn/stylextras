## Accessibility and browser behavior

Each part renders and styles the native element named above. The API reference states the exact React or inherited prop base, including whether `ref` is accepted, every omitted native prop, and every component-applied default. Any native prop not named in `Omit` remains accepted, including standard ARIA/data attributes and event handlers. `sx` is composed last.

The trigger supports <kbd>Enter</kbd>, <kbd>Space</kbd>, <kbd>ArrowDown</kbd>, and <kbd>ArrowUp</kbd>. The menu uses `focusgroup` for wrapped arrow-key navigation plus <kbd>Home</kbd> and <kbd>End</kbd>; a feature-detected polyfill supplies that behavior where necessary. Printable keys move focus by item label, <kbd>Escape</kbd> closes the menu and restores trigger focus, and <kbd>Tab</kbd> closes it after focus moves out.

Disabled items use `aria-disabled` so they remain discoverable by arrow-key navigation while activation is suppressed. `aria-controls`, `aria-expanded`, and the menu's accessible name are synchronized with the native popover state.

### DropdownMenuTrigger

`DropdownMenuTriggerProps`

Complete native/inherited prop base: `Omit<ButtonProps, "popoverTarget" | "popoverTargetAction">`. The props below complete `DropdownMenuTriggerProps`.

- `aria-haspopup` — `DropdownMenuTriggerProps["aria-haspopup"]`. Default: `'menu'`.
- `aria-controls` — `DropdownMenuTriggerProps["aria-controls"]`. Default: `target`.
- `aria-expanded` — `DropdownMenuTriggerProps["aria-expanded"]`. Default: `false`; synchronized with the popover's open state.
- `id` — `string`. Default: `` `${target}-trigger` ``.
- `size` — `ButtonSize`. Default: `'md'`.
- `sx` — `stylex.StyleXStyles`.
- `target` — `string`. Required.
- `type` — `DropdownMenuTriggerProps["type"]`. Default: `'button'`.
- `variant` — `ButtonVariant`. Default: `'outline'`.
