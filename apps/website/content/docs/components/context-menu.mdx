### Example code

```tsx
import * as stylex from '@stylexjs/stylex';
import { colors } from '../tokens/color.stylex';
import { radius } from '../tokens/radius.stylex';
import { spacing } from '../tokens/spacing.stylex';
import { stroke } from '../tokens/stroke.stylex';
import {
  ContextMenu,
  ContextMenuButton,
  ContextMenuItem,
  ContextMenuLabel,
  ContextMenuSeparator,
  ContextMenuTrigger,
} from './index';

const menuId = 'canvas-context-menu';

export default function Example() {
  return (
    <>
      <ContextMenuButton target={menuId}>Canvas actions</ContextMenuButton>
      <ContextMenuTrigger aria-label="Canvas actions" target={menuId} sx={styles.target}>
        Open the context menu anywhere in this area
      </ContextMenuTrigger>
      <ContextMenu id={menuId} aria-label="Canvas actions">
        <ContextMenuLabel>Canvas</ContextMenuLabel>
        <ContextMenuItem>Paste</ContextMenuItem>
        <ContextMenuItem>Duplicate</ContextMenuItem>
        <ContextMenuSeparator />
        <ContextMenuItem>Delete</ContextMenuItem>
      </ContextMenu>
    </>
  );
}

const styles = stylex.create({
  target: {
    borderColor: colors.border,
    borderRadius: radius.sm,
    borderStyle: 'dashed',
    borderWidth: stroke.thin,
    padding: spacing.xxl,
  },
});
```

## Anatomy

- `ContextMenu` — Styled `<div>`.
- `ContextMenuButton` — Visible `<button>` alternative to the context gesture.
- `ContextMenuItem` — Styled `<button>`.
- `ContextMenuLabel` — Styled `<div>`.
- `ContextMenuSeparator` — Styled `<hr>`.
- `ContextMenuTrigger` — Focusable, named `<div role="group">` context-gesture region.

## Accessibility and browser behavior

Give both the trigger region and menu an accessible name. Pointer users can right-click or long-press the region; keyboard users can press the Context Menu key or Shift+F10. Always render `ContextMenuButton` as the visible keyboard and touch alternative when the context action is not available elsewhere.

Disabled items remain arrow-key focusable so their labels are discoverable, but cannot be activated. Escape closes the menu and restores focus to its invoker. `sx` is composed last.

This component starts with semantic HTML and enhances it with newer browser APIs when they are available. When the relevant browser interaction API is unavailable, the package loads a narrowly scoped, feature-detected bridge for that behavior.

## API reference

### ContextMenu

`ContextMenuProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"div">, "aria-label" | "aria-labelledby" | "className" | "popover" | "role" | "style">`. The props below complete `ContextMenuProps`.

- `sx` — `stylex.StyleXStyles`.
- One of `aria-label` or `aria-labelledby` is required.

### ContextMenuButton

`ContextMenuButtonProps`

Complete inherited prop base: `AccessibleButtonPropsWithout<"aria-controls" | "aria-haspopup" | "popoverTarget" | "popoverTargetAction">`.

- `target` — `string`. Required.
- Icon sizes require `aria-label` or `aria-labelledby`.

### ContextMenuItem

`ContextMenuItemProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"button">, "className" | "role" | "style">`. The props below complete `ContextMenuItemProps`.

- `sx` — `stylex.StyleXStyles`.
- `type` — `ContextMenuItemProps["type"]`. Default: `'button'`.

### ContextMenuLabel

`ContextMenuLabelProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"div">, "className" | "style">`. The props below complete `ContextMenuLabelProps`.

- `sx` — `stylex.StyleXStyles`.

### ContextMenuSeparator

`ContextMenuSeparatorProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"hr">, "className" | "style">`. The props below complete `ContextMenuSeparatorProps`.

- `sx` — `stylex.StyleXStyles`.

### ContextMenuTrigger

`ContextMenuTriggerProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"div">, "aria-label" | "aria-labelledby" | "className" | "role" | "style">`. The props below complete `ContextMenuTriggerProps`.

- `aria-controls` — Set to `target`.
- `aria-haspopup` — `ContextMenuTriggerProps["aria-haspopup"]`. Default: `'menu'`.
- `role` — Fixed to `'group'`.
- `tabIndex` — Default: `0`.
- One of `aria-label` or `aria-labelledby` is required.
- `sx` — `stylex.StyleXStyles`.
- `target` — `string`. Required.
