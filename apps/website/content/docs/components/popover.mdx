### Example code

```tsx
import { Popover, PopoverClose, PopoverTrigger } from './index';

const compactId = 'compact-details-popover';
const standardId = 'standard-details-popover';
const wideId = 'wide-details-popover';

export default function Example() {
  return (
    <>
      <PopoverTrigger target={compactId}>Quick details</PopoverTrigger>
      <Popover id={compactId} size="sm">
        <strong>Quick details</strong>
        <span>Short supporting content belongs here.</span>
        <PopoverClose target={compactId} size="sm">
          Close
        </PopoverClose>
      </Popover>

      <PopoverTrigger target={standardId}>View activity</PopoverTrigger>
      <Popover id={standardId} placement="end">
        <strong>Recent activity</strong>
        <span>Three component examples were updated today.</span>
        <PopoverClose target={standardId} size="sm">
          Close
        </PopoverClose>
      </Popover>

      <PopoverTrigger target={wideId}>Review changes</PopoverTrigger>
      <Popover id={wideId} placement="top" size="lg">
        <strong>Ready to publish</strong>
        <span>
          Tokens, examples, accessibility notes, and browser fallbacks are
          included.
        </span>
        <PopoverClose target={wideId} size="sm">
          Dismiss
        </PopoverClose>
      </Popover>
    </>
  );
}
('use client');
```

## Anatomy

- `Popover` — Styled `<div>`.
- `PopoverClose` — Styled `<button>`.
- `PopoverTrigger` — Styled `<button>`.

## Accessibility and browser behavior

Each part renders and styles the native element named above. The API reference states the exact React or inherited prop base, including whether `ref` is accepted, every omitted native prop, and every component-applied default. Any native prop not named in `Omit` remains accepted, including standard ARIA/data attributes and event handlers. `sx` is composed last.

This component starts with semantic HTML and enhances it with newer browser APIs when they are available. Without anchor positioning support, the component remains usable with a simpler non-anchored layout.

## API reference

### Popover

`PopoverProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"div">, "className" | "popover" | "style">`. The props below complete `PopoverProps`.

- `behavior` — `"auto" | "manual" | "hint"`. Default: `'auto'`.
- `placement` — `PopoverPlacement`. Default: `'bottom'`.
- `size` — `PopoverSize`. Default: `'md'`.
- `sx` — `stylex.StyleXStyles`.

### PopoverClose

`PopoverCloseProps`

Complete native/inherited prop base: `Omit<PopoverTriggerProps, "action">`. The props below complete `PopoverCloseProps`.

- `size` — `ButtonSize`. Default: `'md'`.
- `sx` — `stylex.StyleXStyles`.
- `target` — `string`. Required.
- `type` — `PopoverCloseProps["type"]`. Default: `'button'`.
- `variant` — `ButtonVariant`. Default: `'outline'`.

### PopoverTrigger

`PopoverTriggerProps`

Complete native/inherited prop base: `Omit<ButtonProps, "popoverTarget" | "popoverTargetAction">`. The props below complete `PopoverTriggerProps`.

- `action` — `"show" | "hide" | "toggle"`. Default: `'toggle'`.
- `size` — `ButtonSize`. Default: `'md'`.
- `sx` — `stylex.StyleXStyles`.
- `target` — `string`. Required.
- `type` — `PopoverTriggerProps["type"]`. Default: `'button'`.
- `variant` — `ButtonVariant`. Default: `'outline'`.
