## Anatomy

- `Avatar` — Styled `<span>`.
- `AvatarFallback` — Styled `<span>`.
- `AvatarImage` — Styled `<img>`.

## Accessibility and browser behavior

`Avatar` exposes a single image semantic and requires an accessible name. Its visual image and initials are descendants of that named image, preventing duplicate announcements. Use `aria-hidden={true}` on the root for a purely decorative avatar. `sx` is composed last.

This component relies on the browser for its behavior, including keyboard, focus, form, and validation semantics where those apply. It does not ship a component behavior controller. If an optional presentation API is unavailable, the underlying native control and semantics continue to work with the browser’s ordinary presentation.

## API reference

### Avatar

`AvatarProps`

Complete native/inherited prop base omits `aria-hidden`, `aria-label`, `aria-labelledby`, `className`, `role`, and `style`. Supply `aria-label`, `aria-labelledby`, or `aria-hidden={true}`.

- `size` — `AvatarSize`. Default: `'md'`.
- `sx` — `stylex.StyleXStyles`.

### AvatarFallback

`AvatarFallbackProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"span">, "className" | "style">`. The props below complete `AvatarFallbackProps`.

- `sx` — `stylex.StyleXStyles`.

### AvatarImage

`AvatarImageProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"img">, "alt" | "className" | "style">`. The props below complete `AvatarImageProps`.

- `alt` — Native image alternative text; the named Avatar root remains the single exposed image semantic.
- `sx` — `stylex.StyleXStyles`.
