### Example code

```tsx
'use client';

import {
  Command,
  CommandEmpty,
  CommandInput,
  CommandItem,
  CommandList,
  CommandStatus,
  CommandTrigger,
} from './index';

const commandId = 'quick-jump-command';

export default function Example() {
  return (
    <>
      <CommandTrigger target={commandId}>Open command menu</CommandTrigger>
      <Command id={commandId} aria-label="Quick jump">
        <CommandInput aria-label="Search commands" autoFocus placeholder="Search commands…" />
        <CommandList>
          <CommandItem value="button-docs" keywords="components actions">
            Open button docs
          </CommandItem>
          <CommandItem value="tokens" keywords="theme variables">
            Jump to tokens
          </CommandItem>
          <CommandItem value="examples" keywords="gallery demos">
            Browse examples
          </CommandItem>
          <CommandEmpty />
          <CommandStatus>
            {(count) => `${count} command${count === 1 ? '' : 's'} available.`}
          </CommandStatus>
        </CommandList>
      </Command>
    </>
  );
}
```

- `CommandStatus` — Visually hidden polite result announcement.
- `CommandTrigger` — Styled `<button>`.

## Accessibility and browser behavior

Each part renders and styles the native element named above. The API reference states the exact React or inherited prop base, including whether `ref` is accepted, every omitted native prop, and every component-applied default. Any native prop not named in `Omit` remains accepted, including standard ARIA/data attributes and event handlers. `sx` is composed last.

This component uses a focused client controller for composite state and keyboard interaction while retaining native elements for focus and form semantics where possible. Its behavior comes from the component controller rather than a browser-feature polyfill.

## API reference

### Command

`CommandProps`

Complete native/inherited prop base: native dialog props with component-owned styles. `aria-label` or `aria-labelledby` is required.

- `onValueChange` — `(value: string) => void`.
- `sx` — `stylex.StyleXStyles`.

### CommandEmpty

`CommandEmptyProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"div">, "className" | "style">`. The props below complete `CommandEmptyProps`.

- `children` — `CommandEmptyProps["children"]`. Default: `'No results.'`.
- `sx` — `stylex.StyleXStyles`.

### CommandInput

`CommandInputProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"input">, "aria-activedescendant" | "aria-autocomplete" | "aria-controls" | "aria-expanded" | "aria-label" | "aria-labelledby" | "className" | "role" | "style" | "value">`. `aria-label` or `aria-labelledby` is required so the combobox has an accessible name. The props below complete `CommandInputProps`.

- `autoComplete` — `CommandInputProps["autoComplete"]`. Default: `'off'`.
- `sx` — `stylex.StyleXStyles`.
- `type` — `CommandInputProps["type"]`. Default: `'search'`.

### CommandItem

`CommandItemProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"button">, "className" | "onSelect" | "role" | "style" | "value">`. The props below complete `CommandItemProps`.

### CommandList

### CommandTrigger
