## Anatomy

- `DatePicker` — Styled `<input type="date">`.

## Accessibility and browser behavior

Each part renders and styles the native element named above. The API reference states the exact React or inherited prop base, including whether `ref` is accepted, every omitted native prop, and every component-applied default. Any native prop not named in `Omit` remains accepted, including standard ARIA/data attributes and event handlers. `sx` is composed last.

DatePicker has no client boundary or component JavaScript. The browser owns entry, validation, keyboard behavior, reset, and form submission. Compose `Calendar` separately when an enhanced calendar view is useful.

## API reference

### DatePicker

`DatePickerProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"input">, "className" | "style" | "type">`. All native date-input props remain accepted.

- `inputId` — Deprecated alias for the native `id` prop.
- `sx` — `stylex.StyleXStyles`.
