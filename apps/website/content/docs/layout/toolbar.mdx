# Toolbar

Use Toolbar to compose page structure without leaking layout-specific CSS into feature components.

The implementation is intentionally small: visual defaults come from token variables, and the public props keep native HTML attributes available where the underlying element supports them.

## Import

```tsx
import { Toolbar } from "@stylextras/ui/toolbar";
```

## Basic usage

Start here when wiring the component into an interface. The demo shows the expected behavior, and the code block shows the smallest useful invocation.

<Suspense fallback={null}>
  <Example />
</Suspense>

```tsx
import { Toolbar } from "@stylextras/ui/toolbar";

<Toolbar>Example</Toolbar>;
```

## Props

The public prop types are exported from `@stylextras/ui/toolbar`. Most components also accept an `sx` prop for custom styles and forward safe native HTML attributes for the underlying element.

```tsx
export type ToolbarProps = Omit<BaseProps, "className" | "style"> & {
  sx?: StyleXStyles;
};
```

## Customize styles

Use the `sx` prop to compose custom styles with the component's token-driven defaults.

```tsx
import * as stylex from "@stylexjs/stylex";
import { colors } from "@stylextras/ui/tokens/color";
import { spacing } from "@stylextras/ui/tokens/spacing";
import { Toolbar } from "@stylextras/ui/toolbar";

<Toolbar sx={styles.emphasis}>Example</Toolbar>;

const styles = stylex.create({
  emphasis: {
    backgroundColor: colors.bgSubtle,
    padding: spacing.md,
  },
});
```

## Complete example

This example shows the component in a fuller composition with the surrounding pieces it usually needs.

```tsx
import { Toolbar } from "@stylextras/ui/toolbar";
import { Button } from "@stylextras/ui/button";
import { IconButton } from "@stylextras/ui/icon-button";

function ToolbarExample() {
  return (
    <>
      <Toolbar>
        <IconButton label="Undo">U</IconButton>
        <IconButton label="Redo">R</IconButton>
        <Button variant="secondary">Preview</Button>
        <Button>Publish</Button>
      </Toolbar>
    </>
  );
}
```

## Grouped composition

Group related items when the collection should scan as a single unit.

```tsx
import { Toolbar } from "@stylextras/ui/toolbar";

<Toolbar>
  <Button>One</Button>
  <Button variant="secondary">Two</Button>
</Toolbar>;
```

## Accessibility notes

Layout components should not replace semantic landmarks. Use nav, main, aside, section, and heading elements where they describe the content.

## Implementation notes

- Styles use token variables from `@stylextras/ui/tokens/*`.
- The implementation lives in `packages/ui/src/toolbar/index.tsx`.
