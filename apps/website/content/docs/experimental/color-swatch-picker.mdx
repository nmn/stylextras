## Anatomy

- `ColorSwatchPicker` — Native `<fieldset>` containing one radio input per color.

## Accessibility and browser behavior

Each part renders and styles the native element named above. The API reference states the exact React or inherited prop base, including whether `ref` is accepted, every omitted native prop, and every component-applied default. Any native prop not named in `Omit` remains accepted, including standard ARIA/data attributes and event handlers. `sx` is composed last.

Native radio inputs provide arrow-key selection, focus, validation, and form submission. The visible legend names the group, and each swatch has a text label in addition to its color.

## API reference

### ColorSwatchPicker

`ColorSwatchPickerProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"fieldset">, "className" | "defaultValue" | "style">`. The props below complete `ColorSwatchPickerProps`.

- `colors` — `string[]`. Default: `defaultColors`.
- `defaultValue` — `string`.
- `getColorLabel` — `(color: string) => string`.
- `legend` — `ReactNode`. Required.
- `name` — `string`. Required.
- `required` — `boolean`. Default: `false`.
- `sx` — `stylex.StyleXStyles`.
