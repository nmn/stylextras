## Anatomy

- `RangeCalendar` — Native `<fieldset>` with two labeled date inputs.

## Accessibility and browser behavior

Each part renders and styles the native element named above. The API reference states the exact React or inherited prop base, including whether `ref` is accepted, every omitted native prop, and every component-applied default. Any native prop not named in `Omit` remains accepted, including standard ARIA/data attributes and event handlers. `sx` is composed last.

The paired native date controls preserve platform entry, constraints, validation, and form behavior without JavaScript. The legend names the range and each field retains its own visible label.

## API reference

### RangeCalendar

`RangeCalendarProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"fieldset">, "className" | "style">`. The props below complete `RangeCalendarProps`.

- `legend` — `ReactNode`. Required.
- `startLabel`, `endLabel` — `ReactNode`.
- `startName`, `endName` — `string`.
- `startDefaultValue`, `endDefaultValue` — `string`.
- `min`, `max` — ISO date strings.
- `sx` — `stylex.StyleXStyles`.
