## Anatomy

- `ColorSlider` — Native `<input type="range">`.

## Accessibility and browser behavior

`ColorSlider` is a zero-JavaScript native range input. The browser supplies keyboard, pointer, validation, and form behavior. Give each slider a visible label or accessible name that identifies its channel, and provide `aria-valuetext` when the number alone does not describe the color value.

The component adds a visible focus outline and minimum pointer target. Any native input prop not named in `Omit` remains accepted, including standard ARIA/data attributes, event handlers, and `ref`. `sx` is composed last.

## API reference

### ColorSlider

`ColorSliderProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"input">, "className" | "style" | "type">`. The props below complete `ColorSliderProps`.

- `max` — `ColorSliderProps["max"]`. Default: `360`.
- `min` — `ColorSliderProps["min"]`. Default: `0`.
- `step` — `ColorSliderProps["step"]`. Default: `1`.
- `sx` — `stylex.StyleXStyles`.
