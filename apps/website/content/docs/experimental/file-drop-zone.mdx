## Anatomy

- `FileDropZone` — `<label>` containing visible label text and a native `<input type="file">`.

## Accessibility and browser behavior

`FileDropZone` keeps the native file input visible, focusable, and labeled. The browser supplies browsing, keyboard, form, validation, and drag-to-input behavior without a custom drop controller. Use `accept` to provide a file-type hint, but validate files again when processing them.

The containing label reflects focus, invalid, disabled, and forced-colors states through CSS `:has()`. Native input props not named in `Omit` remain accepted, including `multiple`, `required`, `form`, standard ARIA/data attributes, event handlers, and `ref`. `sx` styles the containing label and is composed last.

## API reference

### FileDropZone

`FileDropZoneProps`

Complete native/inherited prop base: `Omit<React.ComponentPropsWithRef<"input">, "className" | "style" | "type">`. The props below complete `FileDropZoneProps`.

- `label` — `ReactNode`. Required.
- `sx` — `stylex.StyleXStyles`.
