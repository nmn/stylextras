## Anatomy

- `EditableText` — Native `<input type="text">` or `<textarea>`.

## Accessibility and browser behavior

Each part renders and styles the native element named above. The API reference states the exact React or inherited prop base, including whether `ref` is accepted, every omitted native prop, and every component-applied default. Any native prop not named in `Omit` remains accepted, including standard ARIA/data attributes and event handlers. `sx` is composed last.

The component uses native form controls instead of `contenteditable`, preserving labeling, selection, paste, validation, reset, and assistive-technology behavior without component JavaScript.

## API reference

### EditableText

`EditableTextProps`

Complete native/inherited prop base: native input props when `multiline` is false, or native textarea props when `multiline` is true. `className`, `style`, and input `type` are component-owned.

- `multiline` — `boolean`. Default: `false`.
- `sx` — `stylex.StyleXStyles`.
